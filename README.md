# qt-notes

一个基于 `Qt 6 Widgets` 的 Linux 便签应用，针对 `Wayland / niri` 的无边框分离窗口场景设计。

## 功能

- 自定义标题栏
- 双击标题重命名
- 标题栏拖动窗口
- 标题栏按钮支持便签列表、主题切换、设置、新建、关闭
- 多个便签分别以独立窗口打开
- 启动时可配置默认打开最后关闭、最后编辑或最后创建的便签
- 便签列表支持多选、全选、批量删除，双击可重命名标题
- 便签列表默认左键在当前窗口切换便签，右键在独立窗口打开
- 便签列表默认按最后编辑时间排序，并支持在设置中改为创建时间或标题排序
- 支持系统字体选择和最近使用字体列表
- 字体与字号为全局设置，修改后所有便签同步生效
- 支持自动换行开关
- 支持 `Ctrl + 滚轮` 调整字号
- 支持粘贴剪贴板图片和拖入本地图片文件
- 图片过大时会自动缩放并限制存储体积，右键支持预览、复制、删除
- 单击图片或右键菜单可打开图片预览对话框，按原始大小内容显示并支持滚动查看
- 支持横向和纵向滚动
- 自动保存文本、主题、字体、换行设置和窗口几何信息
- 设置中可删除当前便签，删除前会二次确认

## 构建

```bash
cmake -S . -B build
cmake --build build -j4
./build/qt-notes
```

## 数据位置

- 数据库：`~/.local/share/snemc/qt-notes/notes.db`
- 图片附件：`~/.local/share/snemc/qt-notes/assets/`
- 应用设置：`~/.config/snemc/qt-notes.ini`

## niri 说明

`qt-notes` 采用自定义无边框窗口，标题栏拖动通过 `startSystemMove()` 实现，适合 Wayland。

如果你希望便签默认以浮动窗口打开，可以在 `niri` 中为 `qt-notes` 配置窗口规则。若后续需要稳定的 `app-id` 匹配，建议额外安装 `qt-notes.desktop`。

窗口大小恢复通常可靠。

窗口位置需要区分平台：

- `X11`：应用会保存并恢复窗口坐标与尺寸
- `Wayland / niri`：应用会保存尺寸和上次所在屏幕信息，但普通顶层窗口不能自行决定精确坐标，最终位置仍由 compositor 控制
